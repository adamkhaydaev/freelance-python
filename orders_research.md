# Анализ реальных заказов (orders_research.md)

Источники: kwork.ru (биржа заказов), fl.ru (раздел проектов).
Цель — разбор живых формулировок заказчиков: что прозрачно, что туманно, какие вопросы задать до старта.

Типы: 1 — Скрипты автоматизации · 2 — Парсинг/сбор данных · 3 — Отчёты и аналитика · 4 — Боты (Telegram/Discord) · 5 — Интеграции

### Заказ 1

- **Источник:** kwork.ru (биржа заказов)
- **Заголовок:** «Скрипты на Python» — помощник для скриптов взаимодействия с ИИ-моделями на Python
- **Тип:** 1 / 5 (скрипты автоматизации с интеграцией API ИИ-моделей)
- **Что в описании прозрачно:**
  - Язык — Python, направление — работа с ИИ-моделями (API нейросетей).
  - Бюджет назван: желаемый до 50 000 ₽, допустимый до 150 000 ₽.
- **Что туманно / отсутствует:**
  - Само ТЗ скрыто («вышлем после рассмотрения резюме») — объём непонятен.
  - С какими именно ИИ-моделями работаем и что скрипты должны делать.
  - Разовая задача или постоянная занятость.
- **Первые 3 вопроса заказчику:**
  1. С какими ИИ-моделями/API работаем (OpenAI, локальные, какие именно)?
  2. Что скрипт должен делать на выходе — какой конкретный результат?
  3. Это разовый скрипт или долгосрочное сотрудничество?
- **Красные флаги:** «ТЗ после резюме» — суть не раскрыта до отбора; очень широкая вилка бюджета (50k–150k) — заказчик сам не определил масштаб; покупатель новый (1 проект на бирже).

### Заказ 2

- **Источник:** kwork.ru (биржа заказов)
- **Заголовок:** «Техническое сопровождение парсера (Python)» — поддержка и доработка существующей real-time системы реверсивного парсинга
- **Тип:** 2 / 5 (парсинг + интеграции: Airtable API и Telegram API)
- **Что в описании прозрачно:**
  - Что есть: работающая система парсинга в реальном времени на 7 клиентов через прокси, данные идут в Telegram и Airtable.
  - Что нужно: стабилизация, устранение ошибок, оптимизация интеграций (batching для Airtable, соблюдение rate limits Telegram).
  - Требования к исполнителю: Python от 3 лет, профильное высшее; только прямой разработчик, без студий и посредников.
- **Что туманно / отсутствует:**
  - Не виден сам код, его объём и качество — оценить трудозатраты вслепую сложно.
  - Не сказано, какие именно «периодические ошибки» сейчас возникают.
  - Формат занятости и оплаты (помесячно, почасово, за задачу).
- **Первые 3 вопроса заказчику:**
  1. Можно ли посмотреть код / архитектуру до оценки (хотя бы стек и объём)?
  2. Какие именно ошибки сейчас повторяются — есть примеры или логи?
  3. Формат оплаты: помесячно за сопровождение, почасово или за конкретные задачи?
- **Красные флаги:** высокие требования (3+ года, вышка) при желаемом бюджете до 50k — возможна несостыковка ожиданий и оплаты; унаследованная чужая система (риск утонуть в чужих багах).

## Сводка

- **Самый частый тип заказов в моей выборке:** Боты (Telegram/Discord) — тип 4. Прямо к нему относятся 4 из 8 заказов (№4, 5, 6, 7), и ещё несколько на стыке с ботами.
- **Самый частый недостаток описаний:** два недостатка встречаются чаще всего — (1) бюджет «по договорённости» / «после собеседования», то есть деньги не названы заранее (№3, 6, 7, 8); (2) просьба чинить чужой код без доступа к нему до оценки (№2, 3, 5, 6). Оценить такие задачи вслепую нельзя.
- **Какой один тип задач я выбираю для себя на ближайшие 2 недели:** Боты (Telegram/Discord) — тип 4.
- **Почему именно этот тип (1–2 строки):** У меня уже есть реальный опыт с ботами на Python (aiogram): делал платного музыкального бота с оплатой через Telegram Stars и автоматическими возвратами. Спрос на ботов в выборке самый высокий, а порог входа для меня ниже, чем в чужих legacy-системах или обходе капч.
