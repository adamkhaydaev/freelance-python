# Price Mirror Bot — Telegram-бот зеркалирования прайса

> Бот берёт прайс на устройства из исходного Telegram-канала, снижает каждую цену
> на фиксированную сумму и публикует обновлённый прайс в основном канале.
> Для продавца, который перепродаёт устройства со своей наценкой вниз.

## Что внутри

- Читает сообщение-прайс из исходного Telegram-канала.
- Снижает каждую найденную цену на заданную в настройках сумму (по умолчанию 1 000 ₽).
- Форматирует прайс в единый аккуратный вид.
- Публикует прайс в основном канале и **редактирует** существующее сообщение, а не плодит новые.
- Обновляет прайс автоматически: при появлении нового прайса в исходном канале либо по таймеру (3 раза в день).

**Что НЕ делает:**
- Не принимает оплату и не выдаёт товар — только публикует прайс.
- Не работает с прайсом в виде картинки/PDF — только текстовое сообщение.
- Не имеет веб-панели: все настройки в файле `config.txt`.

## Установка

```bash
# 1. Распаковать архив и зайти в папку
cd price-mirror-bot

# 2. Создать виртуальное окружение
python -m venv venv

# 3. Активировать его
#    Windows:
venv\Scripts\activate
#    Linux/Mac:
source venv/bin/activate

# 4. Установить зависимости
pip install -r requirements.txt
```

## Запуск

```bash
# Перед первым запуском впишите свои данные в config.txt (см. ниже)
python bot.py
```

Бот запустится, подключится к каналам и начнёт следить за исходным прайсом.
Для постоянной работы держите его запущенным на сервере (см. «Ограничения»).

## Входные данные

Все настройки — в файле `config.txt` (создаётся из `config.example.txt`):

```
BOT_TOKEN=123456:ABC-токен_от_BotFather
SOURCE_CHANNEL=@source_prices      # откуда брать прайс
TARGET_CHANNEL=@my_shop_prices     # куда публиковать
PRICE_DELTA=1000                   # на сколько ₽ снижать каждую цену
UPDATE_MODE=auto                   # auto (по новому прайсу) или timer (3 раза в день)
```

- **Обязательно:** `BOT_TOKEN`, `SOURCE_CHANNEL`, `TARGET_CHANNEL`.
- **Опционально:** `PRICE_DELTA` (по умолчанию 1000), `UPDATE_MODE` (по умолчанию auto).

Исходный прайс должен быть **текстовым** сообщением, цена — числом с ₽ или без
(например: `iPhone 15 — 75 000 ₽`).

## Выходные файлы

- `bot.py` — основной скрипт бота.
- `config.example.txt` — образец настроек (свои данные вписываются в `config.txt`).
- `requirements.txt` — список библиотек.
- `bot.log` — лог работы (когда обновлял прайс, какие ошибки). Растёт со временем, ~1–5 КБ в день.
- `README.md` — этот файл.

## Логика обработки (1 абзац)

Бот подписан на исходный канал. Когда там появляется новый прайс (или срабатывает
таймер), бот берёт текст, находит в нём все цены, вычитает из каждой заданную сумму,
собирает прайс заново в едином виде и обновляет одно закреплённое сообщение в вашем
канале. Старое сообщение редактируется, новые не плодятся, так что канал остаётся чистым.

## Ограничения и допущения

- Бот работает, **только пока запущен**. Для режима 24/7 нужен сервер или VPS
  (на обычном выключаемом ПК бот остановится вместе с компьютером).
- Предполагается, что прайс приходит текстом. Если в исходном канале пришлёт картинку —
  бот её пропустит и запишет в `bot.log` строку «прайс не распознан (не текст)».
- Telegram ограничивает частоту запросов (rate limits). При частых обновлениях бот
  выжидает паузу — это нормально, видно в логах.
- Если Telegram недоступен, бот делает повторные попытки и пишет об этом в `bot.log`,
  не падая.

## Что НЕ входит в эту поставку

- Распознавание прайса с картинок/PDF (только текст).
- Веб-панель или админка для управления — настройки правятся в `config.txt`.
- Размещение и настройка сервера/VPS — отдаётся код и инструкция, развёртывание отдельно.
- Приём оплаты, выдача товара, работа с покупателями.

## Поддержка

- Гарантийное окно: **14 дней** после сдачи.
- Входит бесплатно: исправление багов в согласованной функциональности (снижение цены,
  публикация, обновление сообщения).
- НЕ входит: новые функции, изменение ТЗ, поломки из-за смены Telegram API или изменения
  формата исходного прайса заказчиком.
- Связь: по чату биржи, ответ в течение рабочего дня.
